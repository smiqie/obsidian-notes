- Grid Layout (Гриды) - совремнная фича в отличие от флексов, позволяющая  создавать сложные сетки. флексы и гриды отлично уживаются вместе, т.е. они не взаимно заминяемые (модно и нужно комбинировать)
- грид контейнер - элемент к которому применены св-во dispaly: grid или dispaly: inline-grid
- грид элементы это прямые дочерние элементы грид контейнера, на которые будут действовать правила грид лайаута
- грид линии - это горизонтальные и вертикальные разделительные линии формируюбщие структуру грид лаяута
- грид ячейка - пространство между соседствующими грид линиями
- грид полоса - пространство ограниченное двумя соседними горизонтальными или вертикальными грид линиями
- грид облатсь - пространство ограниченное 4мя грид линиями
-  по умолчанию размеры грид контейнера как и у блочного, ширина занимает всю доступную, высота под контент = display: grid
- display: inline-grid; - в таком случаем грид контейнер в плане размеров, юудет вести себя как строчный размеры по ширине и высоте будут зависеть от внутреннего содерждимого
- `grid-tempolate-columns: auto auto auto` - можно задать кол-во и размер колонок на которое будет поделено пространство, при этом если кол-во грил элементов будет больше чем заданное кол=во колонок, ьто элементы будут переноситсья на некст строку (задается у самого контейнера?). формато взначений множество, нпр 1. через пробел перечислить разщмер каждой из колонок: auto - т.е. ширина кажого дочернего шрид элемента будет вычеслена автоматически в зависимости от ширины внутреннго содержимого. или px, % etc
- мождем прописывать медлу ширин колнок в квадратных скобках прописывать имена для кадлой грил линии (эти имена можно уведеть в devtools - layout - Show line names (вместо show line numbers) + в grid overlays выбрать необходимый на гридл контейнер )
 ```css
 grid-template-columns:
	 [first] 50px [second] 70% [third] auto [fourth];
	 /* может быть полезно в сложных грид сетках для повышения читаемости кода и лучшгей ориентировки за что отвечает та или иная грид сетка */
	 
	 /* также именованные грид линии могут быть использованы в св-вах позиционирования грид элементов*/
 ```

- `grid-tempolate-columns: repeat(3, 150px);` - также можем задать занчения так, через фкнкцию repeat(кол-во повторений, значение которое будет повторяться укавзанное кол=во раз). `grid-tempolate-columns: repeat(3, 150px); == grid-tempolate-columns: 150px 150px 150px;`
- можем использовать неоднократно функцию repeat(): `grid-tempolate-columns: repeat(2, 150px) repeat(3, 50px); == grid-tempolate-columns: 150px 150px 50px 50px 50px;`
- существует новая особенное единица измерения для грид лаяута: `fr - frection (дробь, доля, часть)`. `grid-tempolate-columns: repeat(3, 1fr )`. с помщью нее указываем то насколько  раных частей будет поделено пространство. в данном случае мы кадой колонке кадому грид элементу задали ширину 1/3 от ширины родителя
- `grid-tempolate-columns: 1fr 1fr 50px` - 3я колонка будет занимать фиксированные 50 пикселей, а 1я и 2я колнки будут делить оставшиеся пространство поровну
- - `grid-tempolate-columns: 2fr 1fr 50px` - 3я колонка будет занимать фиксированные 50 пикселей, остальное пространство браузер поделит на 3 равные части - первые 2 части отдаст 1й колонке а оставшиюся распределит под 2ю колонку
- еще один способ задать значение через функцию `minmax(мин, макс) - диапазон размеров от минимального до максимально возможного значения`. `grid-template-columns: minmax(200px, 35%) 1fr 50px` -  в таком случае браузер сам будет регулировать ширину 1го столбца которая будет варироваться от 200 пикселей до 35% от ширины грид контекйнера


- указания высоты строк: `grid-template-rows`. офрматы задания значений аналогичны grid-tempalte-columns. нпр `grid-template-rows: 40px 80px` - первым двум строкам задали фиксированнуб выосту, остальные же строки буду

- `grid-auto-rows` задание конкретной высоты для строк грид контейнера (редко но может понадобиться).
	1. `grid-template-rows: 40px 80px;` - высота первых 2х строк `grid-auto-rows: 15px;` - высота всех последующих строк
	2. `grid-template-rows: 40px 80px;` - высота первых 2х строк `grid-auto-rows: 15px 30px;` - можем указать несколько значений через пробел, в итоге мы задаим опредленный повторяющийся паттерн по высоте строк
	   

- `grid-auto-columns` аналогичное свойства но для задания паттерна размера колонок --> рабоать будет ток в комбинации с еще одним св-вом `grid-auto-flow: column --> говорит, что элементы грид контейнера не должны переноситсья автоматически на новые строки и чтобы они умещались в одну строку по указанному в grid-auto-columns паттерну с учетом значения св-ва грид темплейт колумнс`
	```css
	.grid-container{
	display: grid;
	gap: 10px;
	/*первая колонка будет занимать 30% от ширины грид контейнера */
	grid-template-columns: 30%; 
	/* а последующие колнки будут занимать ширины по паттерну */
	grid-auto-columns: 50px 100px;
	grid-auto-flow: column;
	}
	```

- `grid-template-areas` - позваоляет указать шаблон сетки грид контйенера. в зачнениии св-ва черезщ пробел указываются строки, в каждой строке перечилсляются названия зон/облатсей через пробел, эти названия могут быть любыми, если нужно указать имя пустой ячейки то нужно использовать символ точки. важно указать имена для каждорый яечйки иначе все рабоать не будет
  ```css
  grid-template-areas:
      "menu catalog advertisement"
      "menu catalog advertisement"
      "menu catalog ."
      "menu order order"
    ;
      
  ```

`grid-area` - после здаания шаблона для грид контенера через св-во grid-template-areas нужно распределить дочерние грид элементы по областям. в зачнении которого указывается имя ячейки из шаблона grid-template-areas (нпр: `grid-area: menu;`)

- свва `gap, column-gap, row-gap` - здаания промежутков для грид лайаута/контенйера. row-gap промежуток по вертикали, column-gap  по горизонатли, gap по вертикали и по горизонтали или обощенно сразщу для двуъ


- выранвиание
1. `justify-content` = (default: normal) // выраниваение по горизщоентальной оси //  start - сетка выравнена по левой стороне грид контейнера // end - сетка прижмет содердимое в конец // center - грид элементы будут находиться по центру грид контейнера //  // stretch - формально аналогично значению по умолчанию normal, при котормо грид элементы будут ратснуты по всей ширине грид контйенера если не указаны фиксированные ширины колонок в grid-template-columns // space-between первая и последняя колнока будут расположены по краям, а промедуточные колонки будут распределеены равномерно в оставшимся пространсве // space-around пространство между колонками будет одинаковым а от краев до колонок в 2 раза меньше // space-evenly расстояние между колонками и от краев до колонок будет одинаковым
2. `align-items` = (default: normal - грид элементв будут занимать сток  выосту скок указано в grid-template-columns в ином случае столько сколько высота их содердимого) // выранивание по вертикальной оси //  end -выравняет элементы по низу 
      //  start -выравняет элементы по верху // center - выравняет элементы по центру //  // stretch -  аналогично значению normal, ратснет элементы на всю выосту грид ячейки/грид строки 
3. для двух этих св-в есть обощенное св-во (шорткат) place-items: <по вертикали/align-items> <по горизонатли/justify-content>; пример `place-items: center end;` или сразу для двух если значения совпадают `place-items: center`


- в
```css
.grid-element{
	grid-column-start: 2; /* задаем номер для стартовой грид линии по горизонатли */
	grid-column-end: 4; /* задаем номер конечной грид линии по горизонатли */
	
	
	/*и аналогично - задают номер начальной и конечной грид линии но уже по вертикали */
	grid-row-start: 1; 
	grid-row-end: 3; 
}


/* МОЖДНО СКОРАТИТЬ используя обощенные св-ва*/

.grid-element{
	grid-column: 2 / 4; /* <grid-column-start> <grid-column-end>*/
	grid-row: 1 / 3; /* <grid-row-start> <grid-row-end>*/
}
```


можем вместо цифр использовать имена грид линий в св-вах grid-column и grid-row + их подсв-в
```css
.grid-container{
grid-template-columns:
	 [start] 1fr [middle-before] 1fr [middle-after] 1fr [end];
}

.grid-element{
	grid-column: middle-before / end; /* такой же результат как и в прошлом примере */
	grid-row: 1 / 3;
}
```

если исходя из сождердимого сетки мы можем задать разделительным грид линиям более понятные имена чем просто цифры то это может повысить удобство управления и понимания сетки

+ в этих св-вах можно юзать ключевое слово `span`, нпр `grid-column: span 2; = грид элемен займет ровно 2 столбца, при этом браузер подберет ему наилучшую позицию в сетке грид контейнера`
- ратсягивание одного элемента
```css

.wide{
	grid-column: 1 / 4;
}
or
.wide{
	grid-column: span 3;
}


or - сямое изящное решение (в прошлых случаях мы должны учитывать колво колонок в грид контейнере)
.wide{
	grid-column: -1 / 1; автоматически растянет элемент на все колонки шгрид контйнера независимо от их кол-ва
}
```

<div class="grid-conatiner">
