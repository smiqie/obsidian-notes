- даты используются в реальной парктике почти в кажом проекте
- создавая экземпляр класса Date мы имеем больше возщмодностей

- `const d = new Date()` создание экземпляра класса Date, без каких либо аргументов. по умолчанию это сеголняшняя дата + часовой пояс...?
- `const d = new Date(0)` создание экземпляра класса Date, с аргумнетом 0 (дата анчнется с 1го января 1970 года)
- `d = Date.now()` возвращает таймщтемп (возвращет целое число - колчисетво милисекунл прошедших с 1го января 1970года)
- `d = new Date(2026, 10)` передаем год и месяц (ОТЧЕТ МЕСЯЦОВ ВЕДЕТСЯ С НУЛЯ, те передавая 1 мы получим не январь а феварль)
	- `d = new Date(2026, 0, 15)` передаем год, месяц, день (15 января 2026г)
	- `d = new Date(2026, 0, 15, 9, 30, 33)` передаем год, месяц, день, и время (часы, минуты, секнды) (15 января 2026г 9:30 утра 33 секунды)
	- `d = new Date(2026, 12, 15)` при образению к 12му меясцу мы получим 2027 год 15 января (т.е. другой год). можем писать больше 12ти месяцев
	- `d = new Date("2026-08-15 09:30")` делем тоже самое но в формате строки
		- или так `d = new Date("08-15-2026")` 
		- или так `d = new Date("2026-08-15T09:30:00")`

- методы и свойства экземпляра клсса Date
	- `d.getTime()` возвращает текущий таймстемп
	-  ДОП: log(d, new Date(0).getTime()) // вернет таймстемп и 0 эт че, скок вообще у лог параметров


 - `const d = Date.parse(2024-08-14T00:00:00)` метод parse возвращает таймстемп. те нпр мы поулчили такое значение от бэкэнжда и теперь нам его нужно распарсить в таймстемп

- сеттеры - все сеттеры всразу же изменяюют наш объект
	- `d.setDate(1)` устанавлиавет чисоо месяца . 1е число меясца (нпр августа)
		- `d.setDate(31)` устанавлиавет чисоо месяца . 31 число месяца (нпр августа)
		- `d.setDate(0)` последний день предыдущего меясца (нпр 31 июля)
		- `d.setDate(-1)` предполсдений? день предыдущего меясца (нпр 30 июля)
	- `d.setHours(0)` устаналивает часы
	- `d.setMinutes(0)`
	- `d.setSeconds(0)`
	- `d.setFullYear(2025)` - устанавливаем полный год 
		- `d.setFullYear(2025, 0, 10)` - устанавливаем год, месяц и день (10е января 2025)

- используется нпр для таймеров (нпр отталкиваясь от сегодняшней даты ставим таймер на месяц чтобы проверить подписку или показать рекламный банер и тд)

UTC vs ISO

- utc (coordinated universal time)
	- стандарт времени используемый по всему миру
	- подходит для отобрадения даты и врмеени в текстовом виде, когда важна читаемость


- iso (international organization for standartization)
	- это международнйы стандрат для представления дат и врмеени
	- предпоочтительно исполтзовать для хранения данных и передачи между системами, т.к. формат строгий и однозанчный () 


```js
const now = new Date();
// что делают эти 2 метода? 
now.toUTCString() --> Tue, 20 Aug 2024 14:57:01 GMT (более читаемый)
now.toISOString() --> 2024-08-20T14:57:01.706Z (более предпочтильный для обмена между сервисами, нпр передачи на бэкэкнд)
```
