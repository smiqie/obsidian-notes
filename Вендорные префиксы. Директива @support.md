- свойства могут быть актуальные, устаревшие, или те которые применять еще рано - новые
- вендорные префиксы, каждый из котоых отвечеат за конкретный раузер
```css
- `-webkit-` → Chrome, Safari, Yandex, последнии версии Opera и microsoft edge  и за неокторые другие браузеры на совное движка webkit или blink
- `-moz-` → Firefox
- `-ms-` → старый Internet Explorer + первые версии microsoft edge 
- `-o-` → старые версии Opera (до 12й вклчително)
  
  exnaple
  -webkit-transition: 0.2s ease; /* вендорные префиксы для св-в */
  displey: -ms-flexbox; /* вендорные префиксы для знаечний св-в*/
  input::-moz-placeholder{color: red;} /* вендорные префиксы для псвелокалссов */
  @-webkit-keyframes animka{// код} /* вендорные префиксы для директив */
```
- св-ва с вендорными префиксами отвечают за релаизацию св-ва в конретном браузере
- порядок сущностей важен: снчаал идут сушности с вендоорными префиксами, потом в самом конце без префиксов
```css
-webkit-transition: 0.2s ease;
-o-transition: 0.2s ease;
transition: 0.2s ease;
```
- вендорные префксы нужны для большей совместимости ксс с браузерами т.к. каждый браузер могут интерпитировать по своему  одну и ту же ксс функциональность
- на данный момент времени большинству ксс сущностей вендорные префиксы не нужны, но бывают новые экспереминатльные св-ва, которые в неокторых браузерах без конкретных префиксов рабоать не будут 
```css
/* example */
-webkit-font-smoothing: antialiased; /* экспереминатльнео св-во */
-moz-font-smoothing: antialiased;
/* font-smoothing - управлет сглаживание шрифта (ьудет работаь ток в браузераз макОС и ток с спецаильными вендорными префиксами) */
```

- инструменты для автоматизации и помозши: canisue, autoprefixer, postcss

- директива @supports - модем воспользоваться елси хотим писать код будущего не смотря ни на что // которая в режиме реального времени в браузере пользователя позволяет проверить поддердивается та или иная фича, и как в синтаксиссе медиа запросов - внтури скобок можем использовать новые ксс фичи не боясь что чтото пойдет не так. однако если фича не поддредивается браузером то нужно написать так нащываем фолбэк - запасной набор св-в которые наверняка будут работаь у всех пользоватлеей даже если итоговые результаты будут хуже чем релаизацией новой фичи, аткой фолбэк нужно прописывтаь перед дерективой supports
```css
/* запасное значение (фолбэк) */
body{
	baxckground: url('...') center/cover no-repeat #000; 
}

/* если браузер поддреивает св-во mask-clip */
@supports (mask-clip: view-box){
	/* ... то применятся внутренние стили */
	body{
		mask-image: url('masks.svg#mask1');
		mask-clip: view-box;
	}
}
```
