- псевдоклассы - спец селекторы к элементам, которые удовлетворяют опредленным условиям (нпр - элемент является вторым элементом среди всех своих соседних или не содержит какой либо класс или эдемент находится в состянии наведения курсора и тд )

# 1 - child
## `:first-child`

- получаем доступ к элементу, если он первый дочерний у своего родителя

```css
li:first-child { color: red; }
```

## `:last-child`

- получаем доступ к элементу, если он последний дочерний у своего родителя

```css
li:last-child { border-bottom: none; }
```

## `:ntn-child(odd)`

- получаем доступ ко всем нечетным эдементам (нпр 1, 3, 5 и тд), у своего родителя

```css
li:ntn-child(odd) { border-bottom: none; }
```

## `:ntn-child(even)`

- получаем доступ ко всем четным эдементам (нпр 2,4,6 и тд), у своего родителя

```css
li:ntn-child(even) { border-bottom: none; }
```

## `:ntn-child(2)`

- получаем доступ к конкретному элементу с начала - нпр второму , у своего родителя

```css
li:ntn-child(2) { border-bottom: none; } /* прописывеам конкретнео число */
```

## `:ntn-last-child(2)`

- получаем доступ к конкретному элементу с конца - нпр второму , у своего родителя

```css
li:ntn-last-child(2) { border-bottom: none; } /* прописывеам конкретнео число */
```

## `:ntn-child(3n)` или `:ntn-last-child(3n)`

- получаем доступ к кадому n-ному элементу (нпр к кажому 3му элементу) , у своего родителя. обратимся ко всем терьтим элементам наичная с тертьего

```css
li:ntn-child(3n) { border-bottom: none; } /* прописывеам конкретнео число  с пристафкой n*/
```

## `:ntn-child(3n + 5)`  или `:ntn-last-child(3n + 5)`

- получаем доступ к кадому n-ному элементу (нпр к кажому 3му элементу) , у своего родителя. обратимся ко всем терьтим элементам наичная с индекса указанного после плюса

```css
li:ntn-child(3n + 5) { border-bottom: none; } /* прописывеам конкретнео число  с пристафкой n + с какого индекса будет начинаться подбор*/
```


## `:ntn-child(-n + 4)`  или `:ntn-last-child(-n + 4)` - нестандартный пример применения

- получаем доступ к первым 4м эдементам у своего родителя 

```css
li:ntn-child(-n + 4) { border-bottom: none; } /* прописывеам конкретнео число  с пристафкой n + с какого индекса будет начинаться подбор*/
```

## `:only-child`

- выбирает элемент, который является единственным дочерним эдементом свего роидетля, редко юзается но позволяет сэкономить десятки строчек джавасркипт костылей

```html
<ul>
<li>Item1</li>
<li>Item2</li>
</ul>

<ul>
<li>единственный элемент</li>
</ul>
```

```css
li:only-child { border-bottom: none; }
```


# 2 - not
## `:not()`

- помогает большое екол-во ксс кода елси парвильно юзать
- инвертирует селектор - выбирает всё, кроме указанного условия

```css
li:not(:first-child) { margin-top: 10px; }

div:not(.is-active) { display: none; } /* нпр есть тултип, при нажатии на который добвляется класс is-active, но по умолчаниюб его нет и он скрыт, эта строчка кода позволяет сэкономить строки кода */

li:not(:first-child):not(:last-child) { margin: 10px 0; }
```

# 3 - псевдоклассы состояния (сотсояние элементов которые проявляются когда пользователь взаимодействует с ними)


## `:hover`

- срабатывает, когда курсор наведён на элемент (на touch-устройствах нестабилен)
- хороший ux тон - добавлять на любые интерактивные элементы (кнокпи. ссылки, поля ввода, чекбоксы, селекты, радио-нопки и тд)

```css
/* комбинация двух псевдоклассов */ 
button:hover:focus { background-color: darkblue; }
```

## `:focus`

- срабатывает, когда элемент получает фокус (с помощью клавиатуры + tab)
- ВАЖНО: почти во всех браузерах кроме сафари и фаерфокс, при клике мышкой на элемент, срабатывает это самое состние фокуса, зачастую это создает негативный ux т.к. получается эдакий эффект залипания, кнопка вроде как нажата, курсор убран но фокус остается - решается с помощью псевдокласса focus-visible  
- работает с интерактивными элементами и элементами с `tabindex`

```css
button:focus { box-shadow: 0 0 5px rgba(0,0,255,0.5); }
```

## `:focus-visible`

- работает также как и `:focus`, но срабатывает ток при взаимодействии с элементом через клавитуру 

```css
button:focus-visible {
  box-shadow: 0 0 5px rgba(0,0,255,0.5); 
}
```

## `:active`

- проялвется в момент клика на элемент, на долю мгновения (или если кликнуть ЛКМ и зажать )

```css
button:active {
  background-color: yellow;
  color: indigo;
}
```

## `:disabled`

- сотсяние интерактивного элемента, когда с ним нельзя провзаимодействовать, т.е. когда на него нельзя нажать (сотяонеи disabled можно добавить через одноименный аттрибут disabled элоементам button, input, textarea, select, option, otpgroup, fieldset). браузер при наличии этого состояние добавить блеждно серый цвет для текста и фона.
- позволяет переопредллить disabled по умолчанию и задать новые стили

```css
button:disabled {
}
```

## `:checked`

- характерно для элементов полей ввода с type="checkbox" и type="radio" или  с одноименным аттрибутом checked (чтобы элемент сразу был в таком сотянии при  загрузщке страницу). порзворляет

```css
input:checked {
} /* обращаемся к паолю ввода с таким сотсоянием*/
  box-shadow: 0 0 5px rgba(0,0,255,0.5); 
```

## `:focus-within`

- срабатывает на родительском элементе, если любой из его потомков получил фокус

```css
form:focus-within {
  border: 2px solid blue;
  box-shadow: 0 0 10px rgba(0,0,255,0.3);
}
```

## `:placeholder-shown`

- срабатывает, когда placeholder видим (т.е. input пустой)
- только для элементов с атрибутом `placeholder` (исчезает при вводе текста)

```css
input:placeholder-shown { border-color: gray; }

input:not(:placeholder-shown) { border-color: green; }
```
