- создание учетной записи под проект (нужно чтобы знал/ли от кого были отпарвлены изменения на удаленный репо)
	1. `git config --global/--local user.name "smiqie"`
	2. `git config --global/--local user.email smiqie9201@gmail.com/"smiqie9201@gmail.com"`
	- ФЛАГИ: --local для одного репо (конкретный проект) /// --global для весх проектов (т.е. вес проекты которые мы отпарвляем на удалнный репо то с ними всегда отпарвится эта учетная запись) /// --system команду прописывать с sudo! применяется ко всем существующим учетным записям т.е. отпарвляя проект на удаленнный репо с определнной учетной записью, то учетка с --system всегда отпарвится с ними

- `git config --list` - отображает наши учетные записи (при отправке любого из нащих проектов на уадленный репо, эти данные отпаврятся вместе с ним)

- `git init` производистя в папке проекта // превращает обьычную папку на нашем пк в полноценный репозиторий гит

- `git status` - текущее состояние нащшего каталога (показыывает какие файо ывходят в коммит а какие нет)

- создание локального коммита:
	1. можем создать 100 локальных коммитов прежде чем отпарвить их все в удаленный репо
	2. `git add` - добавление файлов коммит
		1. `git add .` - доавбляет все файлы
		2. `git add cdd/` - доавбляет  конркетную папку
		3. `git add index.html` - доавбляет конкретный файл
	3. `git commit -m "comment"` - локальное сохранение проекта (коммит?)
		- `-m "comment"` - добалвяет коммент к коммиту
		- чекнуть правильный нейминг коммитов!

- создание локального репо + подключение к удаленному и их объедение:
1. `git init`
2. `git commit`
3. `git branch -M main` - переименовывеам главную ветку master  в main
4. `git remote add origin https://ссылка` - подклбчение к удаленному репо // origin = ссылка, которыю мы сразу прописали после
5. `git remote -v` показывает адрес репозитория (fetch/push) // это проверка подключения? 
6. `git push -u origin main` отправляем локальный репо в удаленный (флаг `-u` писать не обязательно?). main - название ветки в которую отправляем  репо . буквально: отправить на origin main // у любого репо всегда есть несколько веток, и начальная ветка всегда main; main - итоговый оттестируемый проект

- `git log` - выводит инфу о коммитах (список всех коммитов)
- `git log -p` - выводит подробную инфу о коммитах (список всех коммитов)
- `git log -3` - выводит инфу о коммитах (флаг -n --> кол-во коммитов которые мы хотим вывести)

- `git clone --depth 1 <url repo>` - клонировать ток послденйи коммит

- `git checkout <hash>` - скачать конкретный коммит по его хэш-коду

- `git rm file.txt` - локальное удаление файла (зачем нужно и поч просто не удалить так: `rm file.txt`?)

- `git rm -r folder` - локальное удаление папки, флаг `-r` - означает рекурсивное удаление (зачем нужно и поч просто не удалить так: `rm -r folder`?)
