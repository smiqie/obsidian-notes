# Objects: Iteration, Destructuring & Methods

**Объекты** в JavaScript предоставляют мощные инструменты для перебора свойств, копирования данных и управления доступом к структуре.

---

# Перебор свойств

## Цикл `for...in`

```js
const user = {
    id: 1,
    name: "Max",
    age: 18,
    position: "UI designer"
};

for (const key in user) {
    console.log(key);       // выводит имя ключа (строка): "id", "name" и т.д.
    console.log(user[key]); // выводит значение по ключу: 1, "Max" и т.д.
}
```

- **Описание:** проходит по всем перечисляемым свойствам объекта.
- **Важно:** на каждой итерации переменная `key` получает имя свойства (строку). Доступ к значению осуществляется через квадратные скобки `user[key]`.

# Деструктуризация и Spread/Rest

## Деструктуризация объекта

```js
const user = {
    id: 1,
    name: "Max",
    age: 18,
    position: "UI designer"
};

const { name, age, ...restProps } = user;

console.log(name);      // "Max"
console.log(age);       // 18
console.log(restProps); // { id: 1, position: "UI designer" }
```

- **Описание:** позволяет извлекать значения из свойств объекта и присваивать их переменным.
- **Особенности:**
    - Порядок не имеет значения (в отличие от массивов).
    - **Rest-оператор (`...`)**: собирает все оставшиеся свойства в новый объект.

---

## Копирование объекта (Spread)

```js
const user = {
    id: 1,
    name: "Max",
    address: {
        country: "USA",
        city: "New-York"
    }
};

// Поверхностное копирование
const newUser = { ...user };

console.log(newUser.address === user.address); // true (ссылка на вложенный объект совпадает)
```

- **Суть:** оператор `...` копирует данные из одного объекта в другой.
- **Мутабельность:** создает **новый** объект, но делает только **поверхностное копирование** (shallow copy). Если внутри есть вложенные объекты, копируется только ссылка на них.

# Объединение объектов

## Оператор Spread vs `Object.assign`

```js
const obj1 = { a: 10, b: 20 };
const obj2 = { c: 30, d: 40 };

// Способ 1: Spread (наиболее современный и лаконичный)
const obj3 = {
    ...obj1,
    a: 100, // перезаписываем значение ключа 'a'
    ...obj2,
    e: 50   // добавляем новое свойство
};

// Способ 2: Object.assign
const obj4 = Object.assign({}, obj1, obj2);
```

- **Важно:** если имена ключей совпадают, значение будет взято из последнего объекта (или перезаписано тем, что указано позже в литерале).

# Статические методы класса Object

## `Object.keys(obj)`

```js
const user = { id: 1, name: "Max" };
console.log(Object.keys(user)); // ["id", "name"]
```
- **Описание:** возвращает массив ключей объекта.

---

## `Object.values(obj)`

```js
const user = { id: 1, name: "Max" };
console.log(Object.values(user)); // [1, "Max"]
```
- **Описание:** возвращает массив значений объекта.

---

## `Object.entries(obj)`

```js
const user = { id: 1, name: "Max" };
console.log(Object.entries(user)); // [ ["id", 1], ["name", "Max"] ]
```
- **Описание:** возвращает массив пар `[ключ, значение]`.

---

## `Object.hasOwn(obj, prop)`

```js
const user = { id: 1, name: "Max" };
console.log(Object.hasOwn(user, "name")); // true
```
- **Описание:** проверяет наличие собственного свойства у объекта (современная замена `hasOwnProperty`).
- **Возвращает:** `true` / `false`.

---

## `Object.freeze(obj)`

```js
const user = { name: "Max" };
Object.freeze(user);

user.name = "Alena"; // действие игнорируется (в strict mode будет ошибка)
delete user.name;    // действие игнорируется
```
- **Описание:** "замораживает" объект, делая его **только для чтения** (readonly). Нельзя добавлять, удалять или изменять свойства.

# Строгий режим (`'use strict'`)

**Строгий режим** делает JavaScript более "безопасным", превращая скрытые ошибки в явные исключения.

```js
'use strict';

// ❌ Ошибка: нельзя создавать переменную без объявления (let/const/var)
age = 10; // ReferenceError

// ❌ Ошибка: дублирование имен параметров
function example(a, a) { // SyntaxError
    return a;
}
```

- **Применение:** прописывается в самом начале JS-файла.
- **В React:** строгий режим включен по умолчанию в модулях и через компонент `<StrictMode>`.
