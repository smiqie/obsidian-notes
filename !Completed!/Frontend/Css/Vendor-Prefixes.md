# Типы CSS свойств

- **Актуальные** - работают во всех современных браузерах
- **Устаревшие** - не рекомендуется использовать
- **Новые/экспериментальные** - применять еще рано (требуют вендорных префиксов)

---

# Вендорные префиксы

```css
-webkit-transition: 0.2s ease; /* для свойств */
display: -ms-flexbox;          /* для значений свойств */
input::-moz-placeholder { color: red; } /* для псевдоклассов */
@-webkit-keyframes animka { /* код */ } /* для директив */
```

- вендорные префиксы отвечают за реализацию css функциональности в конкретном браузере, они нужны для большей совместимостью с разными браузерами 
	- т.к. каждый браузер может интерпретировать CSS функциональность по-своему

**Основные префиксы:**

- `-webkit-` - Chrome, Safari, Yandex, последние версии Opera и Microsoft Edge (браузеры на движке WebKit или Blink)
- `-moz-` - Firefox
- `-ms-` - старый Internet Explorer + первые версии Microsoft Edge
- `-o-` - старые версии Opera (до 12-й версии включительно)

---

# Порядок написания

```css
-webkit-transition: 0.2s ease;
-o-transition: 0.2s ease;
-moz-transition: 0.2s ease;

transition: 0.2s ease;
```

- **порядок важен:** сначала идут сущности с вендорными префиксами, потом в самом конце без префиксов

---

# Когда нужны вендорные префиксы

- на данный момент **большинству CSS свойств вендорные префиксы НЕ нужны**
- нужны для **новых экспериментальных свойств**, которые в некоторых браузерах без конкретных префиксов работать не будут

**Пример экспериментального свойства:**

```css
-webkit-font-smoothing: antialiased;
-moz-font-smoothing: antialiased;
/* работает ТОЛЬКО в браузерах macOS и ТОЛЬКО с вендорными префиксами */
```

# `@supports` - проверка поддержки свойства

```css
/* Запасное значение (фолбэк) */
body {
  background: url('...') center/cover no-repeat #000;
}

/* Если браузер поддерживает свойство mask-clip */
@supports (mask-clip: view-box) {
  /* применятся внутренние стили */
  body {
    mask-image: url('masks.svg#mask1');
    mask-clip: view-box;
  }
}
```

- директива `@supports` - позволяет в режиме реального времени в браузере пользователя проверить поддерживается ли та или иная фича
- **фолбэк** (fallback) - запасной набор свойств, которые наверняка будут работать у всех пользователей (даже если результат хуже чем с новой фичей)
- фолбэк нужно прописывать **перед** директивой `@supports`
