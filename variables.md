- css перменные = токены = кастомные св-ва
- css перменные позволяют хранить какието значения чтобы потом их переисользовать
- использвоание css перменных:
  ```css
  /* объявление css перменных, происходит в рамках каокго-то селектора */
  /* нужно написать 2 символа дефиса, а потмо само название */
  body{
	  /* регистр может быть лююый */
	  --color-main: #20855e; /* кебабкис - использвоать для большинтва */
	  --color-accent: red; /* camel case - использовать для некоторых */
	  --BORDER_RADIUS: 10px;
  }
  button{
	  background-color: var(--color-main);
	  /* исползование значения перемнной чреез функцию var + в скобках имя перменной */
	  
  }
  /* в var обычно исопльзуют оно знаечние, но мы также можем прописать запасное через зпятую, которое будет применено в случае если не сработает первое // не срабоать значение перемнной моет ток в 2х случаях: если перемнная не объялвена вовсе или объявлена но в недостпуной облатси видимости*/
  .btn{
	  box-shadow: var(--box-shadow, 2px 2px 2px green);
  }
  ```
- облатсь видимости перемнных
1. при объялвении перемнной мы привзяывеам ее к определнноум селектору и эта перемнная станвовится доступна для использования текущим элементам и всеми внтуренними элементами

- для чего нужны перемнные?
1. при осомтре макета мы можем выявить повторяюшиеся паттерны - цвета, тени, закругления, отступы и тд

- если речь идет о глобальных параметрах проекта, о глобальных перемнных досутпных по всему преокты, то их принято писать в спецальном селекторе-псевдоклассе root
```css
/* тоже самое что и html{} */
/* перемнные, объявленные в этом селекторе - будут досупны во всем проекте в любом элементе т.к. все элементы находятся в теге html */
:root{

}
/* разница :root и html - у :root будет больше специфичность слектора */
```

- ситуция - на странице есть элемент у которого на фон через backgrund-image установлено изображение
```html
<section class="banner">
	<h1 class="banner-title">Banner Title</h1>
</section>
```
```css
.banner{
	min-height: 100vh;
	background: url('./bg.jpg') center/cover no-repeat;
}
```
если мы разрабатываем обычный сайт (не single page application!) и сайт работает на какой-то CMS то мы можем упростить работу бэкэндеру по предосталвения контент менеджера функционала по изменению этой самой фоновой картинке элемента, для этого оыбчнов хтмл разметки обычно доавбляют аттрибут style где переопределяют то самое св-во backgrund-image т.е. добавляют так называемое инлайновые стили - приоритет таких стилей максимальный 
```html
<section class="banner" style="background-image: url('./bg.jpg');">
	<h1 class="banner-title">Banner Title</h1>
</section>
```
```css
.banner{
	min-height: 100vh;
	background-position: center;
	background-size: cover;
	background-repeat: no-repeat;
	
	...и если мы захотим на урвоне ксс файла переопределить значение св-ва backgrund-image не внося при этом правок в хтмл разметку, то мы вынуждены будем прибешнуть к ключевому слову !important - что зачастую явлется плохой практкиой
	background-image: url('./bg.jpg') !important;
}
```


изяшное решение этой задачи с помощью css перемнных:

```html

вместо св-ва background-image можем указать имя перменной
<section class="banner" style="--bgImg: url('./bg.jpg');">
	<h1 class="banner-title">Banner Title</h1>
</section>
```
```css
.banner{
	min-height: 100vh;
	background-position: center;
	background-size: cover;
	background-repeat: no-repeat;
	
	...затем использовать ее в ксс
	background-image: var(--bgImg); 
	или 
	background-image: none;
	background-color: red;
}
```
теперь если в будущем  нам потребуется повлиять на фоновое изображение на уровне ксс кода, нпр вовсе его убрать или заменить на фоновую заливку сплошным цветом то нам достаточно будет не использовать перемнную фактичсеки объявленную на уровне хтмл разметки
