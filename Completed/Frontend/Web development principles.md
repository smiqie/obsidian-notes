## 1. Responsive Design (Адаптивный дизайн)

- **Верстка отзывчивых элементов** (отзывчивая верстка)
- **Использование медиа-запросов**
- **Использование относительных единиц измерения** (`%` / `vh-vw` / `rem` - `em`)
- **Настройка отзывчивости у изображений**

---

## 2. Обновляемый и понятный код

- **Легкий для понимания другими разработчиками**
- **Независимый и легко дополняемый код**
- **Код, который можно переиспользовать** (даже в других проектах)
- **Правильно организованная система папок и файлов**
- **Использование понятных всем классов**
- **Чистый код с соблюдением всех отступов и стандартов внешнего вида**

💡 **Если поддерживать этот принцип, то в будущем можно брать независимый код из старых проектов в новые проекты**

---

## 3. Минимизация и работа с кодом

- **Как можно меньше HTTP-запросов**
- **Как можно меньше кода**
- **Код сжатый** (минификация)
- **Сжатые изображения**
- **Использование автоматизирующих NPM-пакетов** (программ, плагинов)
	- **Не только NPM, а в целом все инструменты, которые упрощают работу** (например, Sass, PostCSS, сборщики)

💡 **Чем меньше HTTP-запросов, тем быстрее загружается сайт**

```html
<!-- плохо - много запросов -->
<link rel="stylesheet" href="reset.css">
<link rel="stylesheet" href="buttons.css">
<link rel="stylesheet" href="forms.css">

<!-- хорошо - один запрос -->
<link rel="stylesheet" href="styles.min.css">
```

```css
/* до минификации */
.button {
  display: flex;
  padding: 12px 24px;
}

/* после минификации */
.button{display:flex;padding:12px 24px}
```

```html
<!-- сжатые изображения (WebP) -->
<picture>
  <source srcset="image.webp" type="image/webp">
  <img src="image.jpg" alt="описание">
</picture>
```
