# Loops

**Циклы** - нужны для многократного повторения участка кода или для работы с последовательностью (например, вывести коллекцию чисел на экран)

---

# `while` - цикл с предусловием

```js
while (условие) {
    // код
}
```

> **Синтаксис:**
> 
> - Выполняется пока условие `true`
> - Проверка условия **перед** каждой итерацией

## Бесконечный цикл `while(true)`

```js
while (true) {
    // код
    if (условие_выхода) {
        break;
    }
}
```

> **Когда использовать:**
> - Хорошо подходит для бесконечных циклов
> 	- Алгоритмы с неопределенным количеством итераций
> 	- Серверные приложения (ожидание события или сообщения)
> 	- Опрос соединения (ждем ответ)

**Пример из реальной разработки:**

```js
// Функция checkConnection() делает опрос каждые 1.5 секунды
// Возвращает статус: 'pending' или 'connected'
while (true) {
    const status = checkConnection();
    if (status === 'connected') {
        break; // выход из цикла
    }
}
```

> **Важно:** используется редко из-за риска утечки памяти (неверное условие выхода)

# `do...while` - цикл с постусловием

```js
do {
    // код
} while (условие);
```

> **Синтаксис:**
> 
> - Выполняется **минимум 1 раз** (даже если условие `false`)
> - Проверка условия **после** каждой итерации

**Пример:**

```js
do {
    // код
} while (условие);
```

> **Не рационально использовать** - обычно выбирают `while`
> 
> По факту это **синтаксический сахар** (разный синтаксис для одного и того же результата)

# `for` - цикл со счетчиком

```js
for (инициализация; условие; шаг) {
    // код
}
```

**Порядок выполнения:**

1. `let i = 0` - инициализация (выполняется **1 раз**)
2. `i < 5` - проверка условия (в начале каждой итерации)
3. `{ код }` - выполнение кода (если условие `true`)
4. `i++` - шаг (в конце каждой итерации)

**Пример:**

```js
for (let i = 0; i < 5; i+=2) {
    console.log(i);
}
// 0, 2, 4
```

> Обычно принято писать букву `i` в роли счетчика

## Вариации синтаксиса

```js
// Без инициализации и шага (аналог while)
let i = 0;
for (; i < 4; ) {
    i++;
}

// Бесконечный цикл (так никто не делает)
for (;;) {
    // код
}
```

> Нет смысла использовать - есть `while`

## Перебор массива

### С начала массива

```js
const cars = ["audi", "mazda", "tesla"];

for (let i = 0; i < cars.length; i++) {
    console.log(cars[i]);
}
// audi, mazda, tesla
```

### С конца массива (в обратном порядке)

```js
const cars = ["audi", "mazda", "tesla"];

for (let i = cars.length - 1; i >= 0; i--) {
    console.log(cars[i]);
}
// tesla, mazda, audi
```

# Ключевые слова

## `break` - выход из цикла

```js
for (let i = 0; i < 10; i++) {
    if (i === 5) {
        break; // выход из цикла
    }
    console.log(i);
}
// 0, 1, 2, 3, 4
```

## `continue` - переход к следующей итерации

```js
for (let i = 0; i < 5; i++) {
    if (i === 2) {
        continue; // пропускаем итерацию
    }
    console.log(i);
}
// 0, 1, 3, 4 (пропустили 2)
```

# Практические примеры

## Перебор массива объектов

```js
const users = [
    { id: 1, name: "Alex", age: 35, position: "manager" },
    { id: 2, name: "Kate", age: 22, position: "designer" },
    { id: 3, name: "Nikita", age: 29, position: "developer" }
];

for (let i = 0; i < users.length; i++) {
    console.log(`Name: ${users[i].name}`);
    console.log(`Age: ${users[i].age}`);
    console.log(`Position: ${users[i].position}`);
}
```

> `users[i]` - элемент массива (объект), обращаемся к свойству через точку

## Вложенные циклы

```js
for (let i = 0; i < 3; i++) {        // внешний цикл (3 итерации)
    for (let j = 0; j < 2; j++) {    // внутренний цикл (2 итерации)
        console.log(`i: ${i}, j: ${j}`);
    }
}

// Вывод:
// i: 0, j: 0
// i: 0, j: 1
// i: 1, j: 0
// i: 1, j: 1
// i: 2, j: 0
// i: 2, j: 1
```

> Букву вложенного цикла обычно называют `j` или `k`
> 
> Может быть сколько угодно, но чаще всего встречаются только 2 вложенных цикла


## Пузырьковая сортировка

### По возрастанию

```js
const array = [4, 43, 2, 1];

for (let i = 0; i < array.length; i++) {
    for (let j = 0; j < array.length - 1 - i; j++) {
        if (array[j] > array[j + 1]) {
            const temp = array[j];
            array[j] = array[j + 1];
            array[j + 1] = temp;
        }
    }
}

console.log(array); // [1, 2, 4, 43]
```

### По убыванию

```js
const array = [4, 43, 2, 1];

for (let i = 0; i < array.length; i++) {
    for (let j = 0; j < array.length - 1 - i; j++) {
        if (array[j] < array[j + 1]) { // изменили знак
            const temp = array[j];
            array[j] = array[j + 1];
            array[j + 1] = temp;
        }
    }
}

console.log(array); // [43, 4, 2, 1]
```

> **Важно:** в реальных проектах никто не пишет пузырьковую сортировку руками (есть встроенные методы)
> 
> Полезно знать как понимание алгоритма
> 
> Называется пузырьковой, потому что наибольший элемент "всплывает" в конец массива
> 
> Хорошо подходит для маленьких массивов

# `for...of` - перебор элементов

```js
const colors = ['red', 'green', 'blue'];

for (const color of colors) {
    console.log(color);
}
// red, green, blue
```

> **Синтаксис:**
> 
> - Перебирает значения массива (или любой итерируемой коллекции)

**Пример с массивом объектов:**

```js
const users = [
    { name: "Alex", age: 35 },
    { name: "Kate", age: 22 }
];

for (const user of users) {
    console.log(`${user.name} - ${user.age}`);
}
// Alex - 35
// Kate - 22
```

# Термины

**Итерация** - одно выполнение тела цикла (один "круг" цикла)

**Синтаксический сахар** - возможность записать логику разными способами и получить один и тот же результат
