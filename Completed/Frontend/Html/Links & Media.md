# `a` - ссылка

```html
<a href="https://example.com" target="_blank" rel="noopener noreferrer">Внешняя ссылка</a>
<a href="about.html">О нас</a>
<a href="#section">Якорная ссылка</a>
<a href="#">Пустая заглушка</a>
<a href="file.pdf" download>Скачать файл</a>
<a href="mailto:email@example.com">Написать письмо</a>
<a href="tel:+79001234567">Позвонить</a>
```

- `<a>` - создаёт ссылку для навигации по сайту или за его пределами ✅ a11y
- **Атрибуты:**
    - `href`
	    - URL
	    - якорь (`#id`)
	    - заглушка (часто используется при тестировании) `#`
	    - открыть почтовой клиент пользователя `mailto:`
	    - звонок по номеру (особенно на мобилках) `tel:`
    - `download` - скачивание файла вместо перехода
    - `target` - где откроется ссылка:
        - `_self` - в той же вкладке (по умолчанию)
        - `_blank` - в новой вкладке. **Использовать для внешних ссылок**, чтобы не уводить пользователя с сайта
    - `rel="noopener noreferrer"` - почти всегда используется вместе с `target="_blank"`:
        - `noopener` - новая вкладка не может управлять текущей страницей/не имеет доступа через `window.opener` (безопасность)
        - `noreferrer` - не передаёт адрес текущей (нашей) страницы на новый сайт

# `img` - изображение

```html
<img src="path/to/image.jpg" alt="Описание изображения" width="800" height="600" loading="lazy">
```

- **Атрибуты:**
    - `src` - путь к изображению (относительный или абсолютный)
    - `alt` - альтернативный текст ✅ a11y, SEO
	    - заполнять только для значимых изображений; для декора оставлять пустым `alt=""`
	    - поисковики индексируют `alt` - по нему изображение находится в поиске по картинкам
    - `width` / `height` - размеры в пикселях. Важно указывать сразу в HTML, чтобы при загрузке не было скачков контента (CLS)
    - `loading`:
        - `lazy` - изображение загружается только при скролле до него (экономит трафик)
        - `eager` - загружается сразу (по умолчанию)
    - `title` - всплывающая подсказка при наведении

# `video` - видео

```html
<video src="video.mp4" controls width="640" height="360" poster="cover.jpg">
  <source src="video.mp4" type="video/mp4">
  <source src="video.webm" type="video/webm">
  <p>Ваш браузер не поддерживает видео.</p>
</video>
```

- **Атрибуты:**
    - `src` - путь к видеофайлу
    - `controls` - добавляет элементы управления (play, pause, volume)
    - `width` / `height` - размеры плеера
    - `poster` - обложка до воспроизведения (иначе первый кадр)
    - `autoplay` - автовоспроизведение после загрузки
    - `autoplay muted` - автовоспроизведение без звука
    - `loop` - зациклить воспроизведение
    - `playsinline` - для мобильных (видео в своих размерах, а не в полноэкранном режиме)
- **Поддержка разных форматов:** используй `<source>` ==(см. конспект)==

- **Атрибуты:**

    - `loop` - зациклить
- **Поддержка разных форматов:** используй `<source>` ==(см. конспект)==

# `source` - резервные источники медиа

```html
<video controls>
  <source src="video.mp4" type="video/mp4">
  <source src="video.webm" type="video/webm">
  Ваш браузер не поддерживает это видео.
</video>

<audio controls>
  <source src="audio.mp3" type="audio/mpeg">
  <source src="audio.ogg" type="audio/ogg">
</audio>
```

- Указывает альтернативные форматы для `<video>` и `<audio>`
- **Атрибуты:**
    - `src` - путь до файла определенного формата
    - `type` - MIME-тип (подсказка браузеру, как читать файл):
        - Видео: `video/mp4`, `video/webm`, `video/ogg`
        - Аудио: `audio/mpeg`, `audio/ogg`, `audio/wav`
- **FALLBACK** - указание резервных источников данных, повышает вероятность воспроизведения
- Браузер выбирает первый поддерживаемый формат
- Текст после `<source>` - сообщение об ошибке, если ни один формат не загрузился

# `iframe` - встраиваемый контент

```html
<iframe
  src="https://www.youtube.com/embed/VIDEO_ID"
  width="560"
  height="315"
  title="Описание видео"
  allow="autoplay; fullscreen"
  allowfullscreen>
</iframe>
```

- Добавляет сторонний виджет на страницу: карты, видео YouTube, виджет погоды и т.д. ✅ a11y
- **Атрибуты:**
    - `src` - путь к встраиваемому контенту
    - `width` / `height` - размеры фрейма
    - `title` - описание содержимого ✅ a11y
    - `allow` - политика разрешений (`autoplay`, `camera`, `microphone` и т.д.)
    - `allowfullscreen` - разрешить полноэкранный режим
	- `loading`:
	        - `lazy` - виджет загружается только при скролле до него (экономит трафик)
	        - `eager` - загружается сразу (по умолчанию)

# `svg` - векторная графика

```html
<!-- Если не нужна стилизация или анимация — лучше через <img> -->
<img src="icon.svg" alt="Иконка">

<!-- Если нужна стилизация или анимация — вставляем SVG напрямую -->
<svg width="24" height="24" viewBox="0 0 24 24">
  <!-- содержимое SVG -->
</svg>
```

- SVG - векторный формат (Scalable Vector Graphics): не теряет качество при масштабировании, весит меньше растровой графики
- Используется для: иконок, логотипов, декоративных элементов, фигур, линий
- Почти всё, что рисует дизайнер в Figma, можно выгрузить в SVG
- Если не планируется стилизация или анимация - лучше вставлять через `<img>`
