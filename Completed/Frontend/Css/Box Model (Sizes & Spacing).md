# `box-sizing` - расчет размеров элемента

```css
box-sizing: content-box;
box-sizing: border-box;
```

- **Значения:**
    - `content-box` - при добавлении padding или border элемент становится больше (по умолчанию)
    - `border-box` - при добавлении padding или border общий размер не меняется
- `box-sizing` влияет только на внутренние размеры элемента: `content`, `padding`, `border`

---

# Блочная модель

![[box-model.png]]

- **content** - область, где отображается содержимое элемента (ширина и высота определяются `width` и `height`)
- **padding** - внутренние отступы от края элемента (border) до содержимого (content)
- **border** - граница элемента
- **margin** - внешние отступы вокруг элемента (от border до соседних элементов или границ контейнера)

**Когда что использовать:**

- **margin** - нужно отделить элемент от других (создать расстояние снаружи блока)
- **padding** - нужно оттолкнуть контент внутрь от границ блока (создать пространство внутри элемента)

---

# `width` и `height` - фиксированные размеры

```css
width: 500px;
height: 300px;
```

- задают фиксированные размеры элемента
- **плохая практика** задавать фиксированные размеры в `px` - контент может измениться и выйдет за рамки блока

---

# `min-width` и `max-width` - ограничение ширины

```css
min-width: 500px;   /* элемент не может быть меньше */
max-width: 1000px;  /* элемент не может быть больше */
```

- **min/max не задают размеры, а ограничивают их** - работают как рамки
- **Значения:**
    - `min-width` - элемент не может быть меньше указанного значения (может растягиваться шире)
    - `max-width` - элемент не может быть больше указанного значения (подстраивается под экран если он меньше)

# `min-height` и `max-height` - ограничение высоты

```css
min-height: 300px;  /* элемент не может быть меньше */
max-height: 600px;  /* элемент не может быть больше */
```

- **Значения:**
    - `min-height` - элемент не может быть меньше указанного значения (если контент больше - элемент растягивается)
        - полезно для гибкости (пример): `min-height: 640px` - блок минимум 640px, но растянется если контента больше
        - зачастую применяется для секций высотой как минимум текущего окна браузера
    - `max-height` - элемент не может быть больше указанного значения (может быть меньше если контент помещается)
        - зачастую используется в комбинации с `overflow-y: auto`

# `padding` - внутренние отступы

```css
padding: 20px;                    /* все стороны */
padding: 10px 20px;               /* верх/низ, право/лево */
padding: 10px 15px 20px;          /* верх, право/лево, низ */
padding: 10px 15px 20px 25px;    /* верх, право, низ, лево */

/* Отдельные стороны */
padding-top: 10px;
padding-right: 15px;
padding-bottom: 20px;
padding-left: 25px;

/* Логические свойства */
padding-inline: 50px;  /* горизонтальные отступы (влево/вправо) */
padding-block: 50px;   /* вертикальные отступы (вверх/вниз) */
```

- задает величину внутреннего отступа от **края элемента (border)** до содержимого **(content)**
- значения не могут быть отрицательными
- по умолчанию входит в общую ширину/высоту элемента (если не используется `box-sizing: border-box`)

**Паттерн:** ^pattern

1. `padding: x` - отступы со всех сторон
2. `padding: x x` - верх/низ, право/лево
3. `padding: x x x` - верх, право/лево, низ
4. `padding: x x x x` - верх, право, низ, лево

---

# `margin` - внешние отступы

```css
margin: 20px;                    /* все стороны */
margin: 10px 20px;               /* верх/низ, право/лево */
margin: 10px 15px 20px;          /* верх, право/лево, низ */
margin: 10px 15px 20px 25px;    /* верх, право, низ, лево */

/* Отдельные стороны */
margin-top: 10px;
margin-right: 15px;
margin-bottom: 20px;
margin-left: 25px;

/* Логические свойства */
margin-inline: auto;   /* центрирование блока по горизонтали */
margin-block: 50px;    /* вертикальные отступы */
```

- задает величину внешнего отступа вокруг элемента (от `border` до соседних элементов или границ контейнера)
- **margin** в отличие от **padding** не является частью самого элемента (не влияет на фактический размер, но влияет на место в потоке)
- значения могут быть **отрицательными** (иногда бывает полезно)
- браузер может **"схлопывать" margin'ы** - если два соседних блока имеют вертикальные отступы, берется большее из них
- `auto` - автоматический отступ (браузер распределяет свободное пространство поровну → элемент по центру)
- отступы задаются **сверху вниз и слева направо** - элемент не может "толкать себя", его двигают соседи

**Паттерн:** (аналогичен padding [[Box Model (Sizes & Spacing)#^pattern]] )

# `aspect-ratio` - соотношение сторон

```css
aspect-ratio: 16 / 9;
aspect-ratio: 1 / 1;
aspect-ratio: auto;

/* Пример */
.video-container {
  width: 100%;
  aspect-ratio: 16 / 9; /* высота подстроится автоматически */
}
```

- задает соотношение сторон элемента (ширина к высоте) + элемент сохраняет пропорции при изменении размера
- полезно для видео, изображений, карточек чтобы сохранить пропорции
- **Значения:**
    - `width / height` (например, `16 / 9`)
    - `auto` - без фиксированного соотношения (по умолчанию)

# `resize` - изменение размеров элемента

```css
resize: none;
resize: both;
resize: horizontal;
resize: vertical;
```

- указывает правило того, как можно изменять размеры элемента перетаскиванием правого нижнего уголка
    - зачастую применяется к элементу `<textarea>`
    - можно применить к любому элементу у которого задан не стандартный `overflow` (например, `hidden`)
- **Значения:**
    - `none` - размеры изменять нельзя (по умолчанию для большинства элементов)
    - `both` - размер можно менять в обе стороны (по умолчанию для `textarea`)
    - `horizontal` - размер можно менять только по горизонтали
    - `vertical` - размер можно менять только по вертикали **(лучший UX для `textarea`)**

# `overflow` - поведение контента, выходящего за границы блока

```css
overflow: visible;
overflow: hidden;
overflow: scroll;
overflow: auto;
overflow: clip;

/* Управление по осям отдельно */
overflow-x: visible;
overflow-y: hidden;

/* Сокращённая запись для двух осей сразу */
overflow: visible hidden;  /* x y */

/* Фикс бага с обрезанием по одной оси */
overflow: visible clip;    /* вместо visible hidden */
```

- определяет, как блочный элемент ведёт себя если его содержимое не влезает в его размеры
- **Значения:**
    - `visible` - содержимое выходит за пределы контейнера и остаётся видимым (по умолчанию)
    - `hidden` - всё, что выпирает за границы контейнера, скрывается. **Самое популярное значение**
    - `scroll` - всё, что выпирает за границы контейнера, скрывается, но у блока **всегда** появляется полоса прокрутки, даже если контент влезает
    - `auto` - полоса прокрутки появляется **только когда контент действительно выпирает, более умное значение**. Лучше использовать вместо `scroll`
    - `clip` - работает как `hidden`, но строже: при `hidden` можно программно прокрутить скрытое содержимое через JS, `clip` этого не позволяет
