# Типы единиц измерения

- **Относительные** - считаются относительно чего-либо (font-size, viewport, родителя)
- **Абсолютные** - не зависят ни от чего, привязаны к физическим единицам (см, дюйм)

---

# `px` - пиксели

```css
width: 300px;
font-size: 16px;
```

- абсолютная единица измерения
- `1px ≈ 1/96 дюйма` (в 1 дюйме умещается 96 точек)
- фактический размер 1px на разных устройствах разный
- **в большинстве случаев, при работе с `padding` придётся оперировать с помощью `px`!**
- **лучше использовать относительные единицы** когда количество контента может меняться
    - не стоит задавать фиксированную ширину элемента в `px` - пусть браузер сам вычисляет ширину

---

# `%` - проценты

```css
width: 50%;        /* 50% от ширины родителя */
height: 100%;      /* 100% от высоты родителя */
font-size: 120%;   /* 120% от font-size родителя */
```

- относительная единица измерения (считается от родителя)
- **Формула:** `(размер ребенка в px / размер родителя в px) × 100 = размер ребенка в %`
- **Особенность с `height`:**
    - работает только если у родителя указана **конкретная высота**
    - **исключение:** если у ребенка `position: absolute`, то высота считается от ближайшего не статического родителя, иначе от высоты браузерного окна

# `em` - относительно font-size

```css
.parent {
    font-size: 16px;
}

.child {
    font-size: 2em;   /* 16px × 2 = 32px  — берёт от родителя */
    padding: 1em;     /* 32px × 1 = 32px  — берёт уже от себя */
}
```

- относительная единица измерения, считается от `font-size` родителя (или самого элемента)
	- это коэффициент на который умножается значение родительского шрифта
- работает не только с текстом (можно использовать для padding, margin и тд)
- **Реальная практика:**
    - в заголовке нужно сделать часть текста больше или меньше

# `rem` - относительно root font-size

```css
/* если у html font-size: 16px (по умолчанию) */
font-size: 1rem;      /* 16px */
padding: 1.5rem;      /* 24px */
```

- относительная единица измерения, считается от `font-size` корневого элемента (`html`)
	- обычно `1rem = 16px` (браузерное значение по умолчанию)
- полезно когда нужно управлять размером всех элементов из одного места
    - например: режим для слабовидящих - спец меню с кнопками для уменьшения/увеличения текста
    - меняя `font-size` у `html`, автоматически изменятся все размеры на странице, заданные в `rem`

- **Для слабовидящих (трюк с 62.5%):**

```css
html {  
	/* (10px / 16px) × 100 = 62.5% */
	font-size: 62.5%; /* 62.5% = 10px */
}
/* 
Теперь 1rem = 10px (удобно считать), и пользователи смогут увеличить шрифт в настройках браузера и ничего не поломается
*/
```

- слабовидящие могут выставить большой шрифт в настройках браузера
	- чтобы это применилось к нашему сайту, в `html` нужно задать `font-size` в `%`, а не в `px`
- **не рекомендуется использовать** если размеры самого маленького и самого большого шрифтов сильно отличаются (нпр 100px и 10px - плохая пропорциональность)

# `vw` - viewport width

```css
width: 50vw;       /* 50% ширины окна браузера */
font-size: 5vw;    /* 5% ширины окна */
```

- относительная единица измерения, `1vw = 1%` ширины viewport (окна браузера)
- учитывает **полосу прокрутки**
- используется не так часто

---

# `vh` - viewport height

```css
height: 100vh;     /* 100% высоты окна браузера */
min-height: 50vh;  /* минимум 50% высоты окна */
```

- относительная единица измерения, `1vh = 1%` высоты viewport (окна браузера)
- на мобильных устройствах `vh` может работать некорректно, поэтому лучше использовать `svh`, `dvh`, `lvh` 
- **Полезно для:**
    - секций на всю высоту окна браузера
    - всплывающих окон и меню поверх основного контента
