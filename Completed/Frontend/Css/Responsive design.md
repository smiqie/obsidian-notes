# Типы верстки

1. **Фиксированная** - жесткие размеры для элементов (нпр., в `px`)
2. **Отзывчивая (резиновая)** - относительные размеры для элементов (в `%`, `vw` и тд.)
3. **Адаптивная** - использование `@media`, `min/max-width`
4. **Отзывчиво-адаптивная** - комбинация отзывчивой и адаптивной (лучший вариант)

---

# Способы адаптации сайта

1. **Скрытие лишних блоков** при уменьшении экрана
    - элементы, которые не несут смысловой нагрузки (нпр., декоративные изображения и украшательства)
    - важные элементы помещаются во всплывающее меню (открывается при клике на кнопку)
2. **Перестройка сеток** - уменьшение количества колонок
3. **Уменьшение внешних и внутренних отступов**
4. **Уменьшение размера текста пропорционально ширине экрана**

# Подходы адаптивной верстки

## Desktop-First

```css
/* Без медиазапроса - стили для десктопа */
.container {
  width: 1140px;
}

/* Планшет */
@media (max-width: 1023px) {
  .container {
    width: 720px;
  }
}

/* Мобилка */
@media (max-width: 767px) {
  .container {
    width: 100%;
  }
}
```

- сначала верстается полноразмерная версия сайта, затем адаптация под мобилки
- **Desktop-First использовать проще!**

## Mobile-First

```css
/* Без медиазапроса - стили для мобилки */
.container {
  width: 100%;
}

/* Планшет */
@media (min-width: 768px) {
  .container {
    width: 720px;
  }
}

/* Десктоп */
@media (min-width: 1024px) {
  .container {
    width: 1140px;
  }
}
```

- сначала реализуется версия под мобильные устройства, затем адаптация под широкие экраны
- выбор подхода зависит от макета (если дизайнер подготовил только мобильную версию - логичнее использовать Mobile-First)

# `@media` - медиа-запросы

```css
@media (max-width: 700px) {
  /* правила для экранов шириной до 700px */
}

@media (min-width: 401px) {
  /* правила для экранов шириной от 401px */
}
```

- директива `@media` позволяет указывать CSS правила для определенных параметров экрана
	- параметры: размеры, плотность пикселей, тип устройства, пользовательские настройки и тд.

# Meta-тег viewport

```html
<meta name="viewport" content="width=device-width, user-scalable=yes, initial-scale=1.0, maximum-scale=1.0, minimum-scale=1.0">
```

- `name="viewport"` - назначение мета-тега (сообщает браузеру, как масштабировать и отображать страницу на разных устройствах)
- **content параметры - правила масштабирования:**
    - `width=device-width` - **самый главный!** ширина страницы = ширине экрана
    - `user-scalable=yes` - возможность пользователю вручную масштабировать страницу (лучше оставлять `yes`! - **good practice**)
    - `initial-scale=1.0` - начальный коэффициент масштабирования
    - `maximum-scale=1.0` - максимальный коэффициент масштабирования
    - `minimum-scale=1.0` - минимальный коэффициент масштабирования
- **ВАЖНО:** без этого тега медиа-запросы не будут действовать, интерфейс будет просто пропорционально масштабироваться

# Медиа-запросы по высоте

```css
.banner-title {
  color: black;
}

@media (min-height: 900px) {
  .banner-title {
    color: green;
  }
}

@media (max-height: 400px) {
  .banner-title {
    color: red;
  }
}
```

- можно ориентироваться на высоту устройства через `min-height` / `max-height`

---

# Медиа-запросы по ориентации

```css
@media (orientation: portrait) {
  /* стили для вертикальной ориентации */
}

@media (orientation: landscape) {
  /* стили для горизонтальной ориентации */
}
```

- проверка текущей ориентации экрана устройства (чаще всего используют на мобильных устройствах)
	- `portrait` - вертикальная ориентация
	- `landscape` - горизонтальная ориентация

# Медиа-запрос для печати

```css
@media print {
  body {
    font-size: 36px;
  }
  a {
    color: black;
    text-decoration: underline;
  }
}
```

- тип устройства - принтер на который выводится печать
	- стили применяются **только когда страница выводится на печать** (на принтер или в окно предпросмотра печати)
- при нажатии `Ctrl+P` в Chrome браузер откроет превью страницы для печати
	- к этой странице будут применены стили из медиа-запроса `print`

# Медиа-запрос для hover

```css
<button>Click</button>

/* Стандартный hover */
button {
  background-color: palegreen;
}

button:hover {
  background-color: crimson;
}
```

**Проблема на touch-устройствах:**

- при нажатии на кнопку (не клике!) цвет покрасится в `crimson` и останется в таком состоянии (залипнет)
	- это плохой UX, потому что на тач-устройствах **hover физически не существует**

**Решение:**

```css
button {
  background-color: palegreen;
}

/* Эффект hover только для устройств с курсором */
@media (hover: hover) {
  button:hover {
    background-color: crimson;
  }
}
```

- `@media (hover: hover)` - стили наведения применяются **только** если устройство поддерживает курсор
	- эффект наведения на touch-устройствах не будет срабатывать

**ВАЖНО:** в DevTools в режиме эмуляции устройств все режимы по умолчанию являются touch-устройствами (без мыши)

# Хорошая практика - миксин для медиа-запроса (SASS)

```scss
@mixin mobile {
  @media (max-width: 768px) {
    @content;
  }
}

.card {
  width: 300px;
  @include mobile { width: 100%; }
}
```

- создание миксина с медиа-запросом упрощает работу
- все медиа-запросы в одном месте (легко изменить breakpoint)
- `@content` - вставляет CSS код, который мы передаём в миксин ==**!!ДОБАВИТЬ В КОНСПЕКТ!!**==

# Примеры готовых breakpoints

```css
/* Desktop-First */
@media (max-width: 1200px) { .container { width: 1140px; } }
@media (max-width: 1199px) { .container { width: 960px; } }
@media (max-width: 991px)  { .container { width: 720px; } }
@media (max-width: 767px)  { .container { width: 540px; } }
@media (max-width: 575px)  { .container { width: 100%; } }
```

- **Минимальная ширина для адаптива:** 360px (меньше не стоит адаптировать)
- **Больше примеров:** https://getbootstrap.com/docs/5.0/layout/breakpoints/
