# `display` + виды элементов

```css
/* Элемент отображается как блочный */
display: block;

/* Элемент отображается как строчный */
display: inline;

/* Элемент отображается как строчно-блочный */
display: inline-block;

/* Элемент полностью скрывается со страницы */
display: none;

/* Элемент становится flex-контейнером */
display: flex;

/* Элемент становится inline flex-контейнером */
display: inline-flex;

/* Элемент становится grid-контейнером */
display: grid;
```

==**display: grid;**==

- **Значения:**
    1. `block`
        - элемент начинается с новой строки
        - занимает всю доступную ширину родителя
        - высота подстраивается под внутренний контент (`height: auto`)
        - можно задавать `width`, `height`, `margin`, `padding` и т.д.
        - может содержать в себе блочные и строчные элементы
        - 💡 Задавать фиксированные размеры (`width`, `height` в `px`) - плохая практика, так как контент может измениться и выйдет за рамки блока
    2. `inline`
        - элемент располагается в одну строку со своими соседями (если не хватает места по ширине - переносятся на новую строку)
        - ширина и высота подстраиваются под контент
        - нельзя задавать `width`, `height` и вертикальные отступы (`margin-top`, `margin-bottom`) - не работают
        - можно использовать горизонтальные отступы (`margin-left`, `margin-right`) и `padding`
        - может содержать в себе только строчные элементы
    3. `inline-block`
        - элементы ведут себя как строчные - располагаются в одну строку
        - при этом можно задавать `width`, `height`, `margin`, `padding`
        - ширина и высота по умолчанию подстраиваются под контент
        - может содержать строчные и блочные элементы
    4. `none`
        - элемент полностью скрывается со страницы
        - не занимает место в потоке (как будто его нет)
        - в HTML-разметке элемент остается (можно снова показать через CSS или JavaScript)
        - используется для скрытия элементов (меню, всплывающих окон, модалок и т.д.)
        - 💡 **Отличие от `visibility: hidden`:**
	        - `visibility: hidden` скрывает элемент, но сохраняет место в потоке, а `display: none` убирает элемент полностью
    5. `flex`
        - элемент становится flex-контейнером, а все дочерние элементы становятся flex-элементами
        - основное преимущество - удобное выравнивание и распределение элементов по горизонтали и вертикали
        - контейнер ведёт себя как **блочный элемент** (занимает всю ширину родителя, начинается с новой строки)
    6. `inline-flex`
        - элемент становится flex-контейнером
        - контейнер ведёт себя как **строчный элемент** (располагается в строку с другими элементами, ширина подстраивается под содержимое)
        - 💡 В 95% случаев используется `flex`
	        - `inline-flex` нужен редко - только когда flex-контейнер должен быть частью строки текста (кнопки с иконками и т.д.)

|Тип элемента|Расположение|Управление размером|Можно вложить|Начинается с новой строки|
|---|---|---|---|---|
|`block`|один под другим|✅ `width` / `height`|любые|✅|
|`inline`|в одну строку|❌|только строчные|❌|
|`inline-block`|в одну строку|✅|строчные / блочные|❌|
|`none`|не отображается|—|—|—|
|`flex`|задаёт гибкий контейнер|✅|любые|✅|
|`inline-flex`|задаёт строчный flex-контейнер|✅|любые|❌|

# `display: flex` - Flexbox

Flexbox - технология CSS для удобного выравнивания и распределения элементов внутри контейнера по одной оси (горизонтальной или вертикальной)

**Оси контейнера:**

- **Главная (основная)** - вдоль которой располагаются дочерние элементы
- **Поперечная (побочная)** - перпендикулярная главной

```css
/* === СВОЙСТВА FLEX-КОНТЕЙНЕРА === */
display: flex;
flex-direction: row;
justify-content: space-between;
align-items: center;
align-content: space-around;
flex-wrap: wrap;
flex-flow: row wrap;
gap: 20px;

/* === СВОЙСТВА FLEX-ЭЛЕМЕНТОВ === */
align-self: flex-end;
order: 1;
flex-grow: 1;
flex-shrink: 0;
flex-basis: 50%;
flex: 0 1 auto;
```

## Свойства flex-контейнера

1. `flex-direction` - задает направление главной (основной) оси
    
    - **Значения:**
        - `row` - слева направо (по умолчанию)
        - `row-reverse` - справа налево
        - `column` - сверху вниз
        - `column-reverse` - снизу вверх

2. `justify-content` - выравнивает элементы по **главной оси**
    
    - **Значения:**
        - `flex-start` - элементы прижаты к началу оси (по умолчанию)
        - `flex-end` - элементы прижаты к концу оси
        - `center` - по центру
        - `space-between` - первый и последний у краёв контейнера, промежутки между элементами равные
        - `space-around` - равные отступы между всеми элементами, кроме первого и последнего (у них отступ от краёв = половине промежутка между элементами)
	        - **в реальной работе применять редко!**
        - `space-evenly` - равные отступы между **всеми элементами**
	        - **в реальной работе применять редко!**

3. `align-items` - выравнивает элементы вдоль **поперечной оси**
    
    - **Значения:**
        - `stretch` - растягиваются на всю высоту контейнера (по умолчанию)
        - `flex-start` - прижаты к началу поперечной оси
        - `flex-end` - прижаты к концу поперечной оси
        - `start` / `end` - сокращённая запись вместо `flex-start`/`flex-end`, адаптируется к направлению языка (RTL/LTR) - полезно для мультиязычных сайтов
        - `center` - по центру поперечной оси
	        - **самое популярное значение!**
        - `baseline` - выравниваются по базовой линии текста
	        - **бывает полезен при вёрстке блоков по типу списка характеристик товара**
	        - ![[baseline.png]]

4. `align-content` - задаёт распределение свободного пространства между **рядами элементов по поперечной оси** (работает только если `flex-wrap: wrap` и рядов больше одного). **В реальной практике применяется редко**
    
    - **Значения:**
        - `stretch` - ряды одинаково растягиваются по контейнеру и занимают одинаковое пространство (по умолчанию)
        - `flex-start` - ряды прижаты к началу поперечной оси
        - `flex-end` - ряды прижаты к концу поперечной оси
        - `center` - все ряды по центру
        - `space-between` - первый ряд у начала, последний у конца, остальные распределены равномерно
        - `space-around` - см. ==`justify-content: space-around`==
        - `space-evenly` - см. ==`justify-content: space-evenly`==

5. `flex-wrap` - задаёт, должны ли элементы переноситься на новую строку при нехватке места
    
    - **Значения:**
        - `nowrap` - все элементы в одну строку (по умолчанию)
        - `wrap` - элементы переносятся на новую строку
        - `wrap-reverse` - переносятся, но в обратном порядке (новые строки сверху) 
	        - **крайне непопулярное значение**

6. `flex-flow` - объединяет `flex-direction` и `flex-wrap` в одно свойство
    
    - Паттерн: `flex-flow: <direction> <wrap>;`

7. `gap` - задаёт промежутки между элементами внутри flex-контейнера
    
    - Паттерн: `gap: <row-gap> <column-gap>;`
    - **Значения:** любые единицы измерения: `px`, `em`, `%` и т.д. (по умолчанию: `0`)
    - Можно задавать отдельно:
        - `row-gap` - промежуток между **рядами**
        - `column-gap` - промежуток между **колонками**

## Свойства flex-элементов

8. `align-self` - переопределяет `align-items` для конкретного элемента
    
    - **Значения:**
        - `auto` - наследует от контейнера (по умолчанию)
        - `stretch` - растягивается на всю высоту
        - `flex-start` - прижат к началу
        - `flex-end` - прижат к концу
        - `center` - по центру
        - `baseline` - по линии текста

9. `order` - задаёт визуальный порядок расположения элемента внутри контейнера
    
    - **Значения:** любое целое число (`0` по умолчанию). Чем больше число - тем позже элемент идёт в порядке, отрицательные значения - элемент будет раньше
    - Не влияет на HTML-разметку, только на визуальное отображение
    - **злоупотреблять не стоит! в реальной практике обычно используют `-1` чтобы переместить элемент в начало и `1` чтобы переместить в конец**

10. `flex-grow` - указывает, может ли элемент растягиваться и в каких пропорциях относительно соседних элементов
    
    - **Значения:** целое число от `0` (по умолчанию: `0` - не растягивается)
    - Элемент будет стараться растянуться, заняв доступное свободное место
    - Работает даже если элементу заданы фиксированные размеры
    - **В реальной практике чаще используют значение `1`, чтобы растянуть элемент на всё доступное пространство**

11. `flex-shrink` - указывает, в каких пропорциях элемент будет сжиматься относительно соседних если в контейнере не хватает места (противоположность `flex-grow`)
    
    - **Значения:** число от `0` (по умолчанию: `1` - сжимается)
    - **в реальной практике используется значение `0`, чтобы элемент не сжимался самопроизвольно, игнорируя свои реальные размеры**

12. `flex-basis` - указывает базовый размер элемента до того, как будет распределено свободное место контейнера за счёт `flex-grow`
    
    - Размеры, заданные через `flex-basis`, всегда приоритетнее `width`/`height`
    - Если `flex-direction: row` - задаёт ширину, если `column` - задаёт высоту
    - **Значения:** любые единицы измерения (`px`, `%` и т.д.). По умолчанию: `auto` (браузер смотрит на `width`/`height` элемента)

13. `flex` - шорткат для `flex-grow`, `flex-shrink`, `flex-basis`
    
    ```css
    /* значения по умолчанию */
    flex: 0 1 auto; /* flex: grow shrink basis */
    ```
    
    - **В реальной практике часто используют:** `flex: 0 0 50%` - элемент не будет ни растягиваться, ни сжиматься, и займёт ровно 50% ширины/высоты flex-контейнера (или конкретный размер в `px`)
