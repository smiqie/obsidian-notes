- единицы измерения
		1. относительные - считаются относительно чего либо
		2. абсолютные - не от чего не зависяти привязаны к физичиским елинам измрения нпр см. или дбймам, 
* доп - 1 viewport = 1% окна браузера (vh & vw)
- vw - относительные // задает размеры в процентах относительно теккущего онка браузера (ширины viewport) + полоса прокуртки (используется не так часто)
- vh - задает размеры в процентах, относителньо высоты viewport, фактичсеки высота текущего окна барузера // viewport единицы часто применяются для разных окон и меню поверх основного контента, а также для секций от 100% высоты окна браузера // могут быть пробелмы на мобилах - лечиться с помозщью svh,dvh,lvh - лучше юбзать их а не vh
- rem - вычисляется относителньо значекния fontsize у корневого элемента (тег html - обычно 1rem=16px) // полеззно там, где планируем управлять размером каких либо элементов из одной тчоки нпр такая необходимость мождет возникнуть когда мы реализовываем режим для соабовидящтз - спец меню с кнопками - для уменьшения и увелечения текста // испольузесря для слабовитядзих (не рекомендуется юзать т.к. часто размеры самого маленького и самого большого шрфитов могут изначально сильно отличаться нпр 100px и 10pz - плохая пропорциональность) // слабовидящие в натсройках барузерах могут выстаивть большойц шрифт, и чтобы к нашему сайту все тож применилось т.е. стили браузера (не было ошибок) и изменился размер в html нужно задать font-size: 10px (удобная штука) --> делаем так (10px/16px-deffault)\*100=62.5% (делаем так тчобы font-size: 10px в html не мешал настрйоках барузера)
- px - лучше юзать относительные единицы измерения когда колво контента в элементе может быть больше тлт меньше вычисления ширины элемента - стоит отдавтаь на откуп браузру ((т.е. не указывать width  в px самоятстельно)! // фактичсекий размер 1px на разных устрйоствах разеый // 1px +-= 1/96 дюйма т.кю в 1 люйм умещается 96 точек
- % - измеряются относительно чегото как правило родителя // (высота ребенка в px / высота родителя в px) \* 100 = выоста ребенка в % // в случае с height  будет работаь елси у родетлоя указана конкретная высота, ичклюбчение: если у ребенка указан position: absolute, то выоста будет расичтыватьсяот ближайщего не статического родителя, иначе от высоты браузерного окна
- em - задает размер относительно размера род шрифта (нпр если у родиетля font-size: 16px; то у ребенка 1em=16px, 0.5em=8px --> font-size (child): 16\*0.5 = 8px ) т.е. это некий коэфицент на который умножается значение род шрфита  // может работаь не ток с тектсом (полезно) // в релаьной практике бывает полезно нпр когда в заголовке нужно было сделать размер какойто его части большим или меньшим - лучше применять em? // указывается в формате коэфицента на который умнождается значение род. шрифта // берет размер шрифта того элемента где применена эта единица либо его родителя?
