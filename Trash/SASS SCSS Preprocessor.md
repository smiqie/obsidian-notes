- @import vs @use vs @forward - чекнуть у ламкова

- архитектура проекта:
1. abstracts --> \_variables.scss (перменные), \_mixins.scss (миксины), \_functions.scss (функции)
2. base (шлобальные стили) --> \_reset.scss // \_normalize.scss, \_animation.scss (для @keyframes), \_base ьазовая натсройка сайат - нпр body{} a{} .container{}? etc, \_typography.scss - стили для текста? нпр заголовки или body + натсройка шрифтов (нужен чтобы текст на всем сайте выгледел одинаково чтобы не прописыватьк ажомуц элементу font-face, шлобальные стили body, h1-h6, a), \_utilites.scss - утилитарные классы нпр .container{} .visually-hidden{}?
3. components папка для компонентов повторяющихся (мелкие повторяюбщие детали - кнопки, карточки и тд) --> нпр \_button.scss
4. layout крупные блоки (нпр для системы грид сеток)--> нпр \_header.scss
5. pages  специфические сили для страницы --> нпр \_home.scss
6. styless.scss - корневой файл, в нем импортируем все сксс файлы

* любоый сксс файл должен иметь имя аничнающиеся с \_ - нужно чтобы сасс компилятор не компилировал автоматически такие файлы а игнорировал их, нужно компилировать ток 1 корневой файл в проекте: \_varisblews.scss - good // variables.scss - bad
