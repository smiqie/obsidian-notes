# Плюсы

1. Использованы семантические теги - header, nav, section, ul - именно то, что требовало ТЗ
2. Карточки меню реализованы как ссылки, а не div - по ТЗ верно
3. Шрифт подключён через Google Fonts
4. Структура проекта соответствует требованиям: отдельный style.css, папка images, нет инлайн-стилей в HTML
5. Сам проект сверстан довольно неплохо для новичка - есть куда расти!

## Баги самого проекта (критические)

### 1. Очень много неиспользуемого и некорректного кода

В проекте обнаружено большое количество неработающего CSS - в основном некорректно заданные значения свойств

Примеры:

```css
/* Не заданы единицы измерения — браузер игнорирует такие значения */
width: 136.8465118408203;
height: 21.288009643554688;
width: 188;
height: 40;

/* Таких значений у font-style не существует.
   Скорее всего перепутано с font-weight.
   Правильно: font-weight: 400 (regular), font-weight: 600 (semi bold) */
font-style: Regular;
font-style: Semi Bold;
font-style: 400;
```

### 2. Пробелы в путях к файлам

Пробелы в пути к файлу недопустимы! - браузер может не найти файл

```html
<!-- Неправильно -->
<img src="./images/Group - logo.png">
```

### 3. Два заголовка первого уровня

В верстке использовано два тега `h1`, что неправильно. **Заголовок первого уровня может быть только один, от него строится иерархия остальных заголовков:**

```html
<h1>Новости</h1>
  <h2>Политика</h2>
  <h2>Технологии</h2>
    <h3>ИИ и разработки</h3>
```

# Рекомендации

1. Хороший тон - закрывать одиночные теги:

    ```html
    <br> --> <br />
    <img> --> <img />
    ```

2. Шрифты лучше подключать локально через `@font-face` в современном формате woff2 - более безопасно и не зависит от внешнего сервиса

3. Названия файлов, папок и импортов должны быть в нижнем регистре - при выгрузке на хостинг сервер может ругаться на регистр. Также рекомендуется давать файлам осмысленные имена (нпр `pizza.png`, `soups.png`), чтобы в будущем не запутаться. Логотип сайта лучше хранить в формате SVG в папке `icons`, как и остальные декоративные элементы не несущие ценности для пользователя

4. Почти все стили реализованы через `id` - это плохой тон. `id` имеют очень высокую специфичность и затрудняют переопределение стилей. Идеальный баланс гибкости и специфичности - `class`. `id` подходит для других целей (нпр. якорных ссылок)

5. Нейминг классов сильно хромает - рекомендуется использовать БЭМ методологию, которая широко применяется в продакшене:
    - [Урок по БЭМ методологии](https://www.youtube.com/watch?v=FH-t1mt-lSs&t=60s)
    - [Как правильно именовать классы](https://www.youtube.com/watch?v=EVWT4exv4jA)

6. Хороший тон - добавлять интерактивным элементам (ссылкам, изображениям и тд.) атрибут `title=""` - служит подсказкой при наведении:
    
    ```html
    <!-- было -->
    <a class="card-menu" href="index.html">
    <!-- стало -->
    <a class="card" href="./index.html" title="Заказать пиццу">
    ```
    
7. Всем изображениям рекомендуется добавлять атрибут `alt` - это в первую очередь доступность для людей с ограниченными возможностями (скринридеры), а во вторых поисковые системы могут найти изображение по этому тексту и лучше продвигают сайты с корректной семантикой и SEO:
    
    ```html
    <!-- было -->
    <img src="./images/Group - logo.png">
    <!-- стало -->
    <img src="./icons/logo.svg" alt="Resto App - Home" title="Resto App - Home" />
    ```
    
8. Логотип в шапке семантически некорректен - картинка без `alt` делает заголовок пустым. Также логотип должен быть ссылкой на главную страницу:
    
    ```html
    <!-- неправильно -->
    <h1 id="logo">
      <img src="./images/Group - logo.png">
    </h1>
    
    <!-- правильно -->
    <a class="logo" href="./index.html" title="Resto App - Home">
      <img class="logo__image" src="./icons/logo.svg" alt="Resto App - Home" />
    </a>
    ```
    
9. Элементы навигации должны быть ссылками, а не просто текстом в `li`:
    
    ```html
    <!-- неправильно -->
    <nav>
      <ul class="ul">
        <li>Меню</li>
        <li>Акции</li>
      </ul>
    </nav>
    
    <!-- правильно -->
    <nav class="main-menu">
      <ul class="main-menu__list">
        <li class="main-menu__item">
          <a href="/menu" class="main-menu__link" title="Перейти в основное меню блюд">Меню</a>
        </li>
        <li class="main-menu__item">
          <a href="/promotions" class="main-menu__link" title="Посмотреть текущие акции и скидки">Акции</a>
        </li>
      </ul>
    </nav>
    ```
    
10. Тег `<br />` лучше заменять ограничителями `max-width` / `min-width` и избегать в верстке
    
11. В проекте нет сброса стандартных стилей - рекомендуется добавить `normalize.css`, `reset.css` или написать свой базовый сброс:
    
    ```css
    *,
    *::after,
    *::before {
      margin: 0;
      padding: 0;
      box-sizing: border-box;
    }
    ```
