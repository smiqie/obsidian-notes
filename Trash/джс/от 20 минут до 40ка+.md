## 1. Объявление переменных 

### Типы объявлений

```js
var x = 10;    // ❌ устаревшее, глобальная видимость (НЕ ИСПОЛЬЗОВАТЬ)
let y = 20;    // ✅ локальная, можно менять
const z = 30;  // ✅ локальная, нельзя менять ссылку (т.к. это константа)
```

- декларирование/объявление?: `let x;`
- присваивание: `let x = 4;` 

- var использует хостинг! (ысплытие наверх при копиляции кода, независимо от того где перменная была обяъвлена)
- лучше всегда использовать константы - это защищает наш код
- перменным нуно давать осмысленные имена!!! - оч сложная наука но важна давать понятные имена

### Хорошие и плохие практики

```js
// ❌ Плохо - неудобно читать
const name='Max', age=28, isDeveloper=true;

// ✅ Хорошо - каждая переменная на отдельной строке
const name = 'Max';
const age = 28;
const isDeveloper = true;
```

### Строгий режим

```js
// ❌ Плохо - создаётся глобальная переменная
age = 10;

// ✅ Хорошо - используйте строгий режим
'use strict'; // прописывать в начале каждого файла JS
```

**Строгий режим** запрещает создавать переменные без `let`/`const`/`var`

## 2. Правила нейминга

**camelCase** - для переменных и функций

```js
let isEnabled = true;
```

**PascalCase** - для классов и конструкторов

```js
class UserAccount {}
```

**SCREAMING_SNAKE_CASE** - для констант

```js
const API_KEY = '123456';
```

- соглашения оменовании пемренных: буквы, цифры, нижние подчеркивание и знак доллара

## 4. Особенности типов данных

### String (строки)

**Кавычки:**

```js
'text'                  // одинарные
"text"                  // двойные
`text ${variable}`      // backticks (обратные кавычки) - позволяют интерполяцию
```

**Интерполяция:**

```js
const name = 'Max';
const a = 5, b = 3;
const msg = `name: ${name}, sum: ${a + b}`; // ✅ читабельно

// ❌ Плохой тон - конкатенация через +
const sum = "sum: " + (a + b);
```

### Number (числа)

**Особенности:**

```js
100 / 0      // Infinity
-100 / 0     // -Infinity
'Word' / 100 // NaN
```

**Диапазон безопасных целых чисел:**

- `Number.MIN_SAFE_INTEGER = -9007199254740991`
- `Number.MAX_SAFE_INTEGER = 9007199254740991`

**Читаемость больших чисел:**

```js
const num = 1_000_000; // ✅ удобно читать
```

### BigInt (большие целые)

Для чисел больше `Number.MAX_SAFE_INTEGER`:

```js
const big = 900n; // ✅

// Операции только с другими bigint
2n + 1;   // ❌ ошибка
2n + 1n;  // ✅ работает
```

- чаще всего использщуется просто Number (BigInt для каикхто узкизх проектов)
### Null (пустое значение)

```js
let data = null; // ✅ переменная пока пустая, ждём значение
```

- Обычно присваивают переменным, значение которых ещё не получено (нпр., с сервера), но вскоре ожидается

### Undefined (неопределённое)

```js
let x; // x === undefined ✅

// ❌ Плохой тон
let name = undefined;

// ✅ Хорошо
let name = null;
```

## 5. Работа с типами

### Оператор typeof

Возвращает строку с типом переменной:

```js
const name = 'Max';
console.log(typeof name); // "string"
```

### Преобразования типов

#### ❌ Неявные (плохой тон)

JS сам пытается привести типы, часто неожиданно:

```js
const num = 111;
const str = '222';
console.log(num + str);  // "111222" - приводит число к строке
console.log('16' / '8'); // 2 - приводит строки к числу
```

#### ✅ Явные (хорошая практика)

Используем глобальные функции:

```js
Number('16');   // конвертирует в number
String(123);    // конвертирует в string
Boolean(0);     // конвертирует в boolean
                // false при: '', null, 0, undefined, NaN
                // true для всего остального
```
