
## 6. Операторы

### Арифметические операторы

`+`, `-`, `/`, `*`, `%`, `**` (возведение в степнеь - нпр `x = 2 ** 3; // 8`), инкремент, декремент

**Постфиксный инкремент/декремент (`a++` / `a--`):**

```js
let a = 10;
let b = a++;
// a = 11
// b = 10
```

**Префиксный инкремент/декремент (`++a` / `--a`):**

```js
let a = 10;
let b = ++a;
// a = 11
// b = 11
```

### Операторы сравнения

`==`, `===`, `!=`, `!==`, `>`, `<`, `<=`, `>=`

**Строгое сравнение (рекомендуется использвоать `!== i ====`, `!= и ==` - устарвешие):**

```js
2 === '2' // false (типы не совпадают)
2 !== 2   // false

// ❌ Избегайте == и != (автоматически приводят типы к единому)
	2 == '2' // true
// ✅ Используйте === и !== (учитывают тип)

true == 1 // true
true === 1 // false
```
- `==` сравнивается с приведением к одному типу
- `===` сранвивается по типу и по значению, ьез приведения к одлнмоу типу

**Сравнение строк:**

⚠️ Строки сравниваются по порядку символов в Unicode

```js
'JavaS' > 'Java'  // true (длина + сравнение позиционно)
'ы' > 'f'         // true (т.к. кириллица идёт позже латиницы в юникоде)
```

### Тернарный оператор

```js
const age = 18;
const status = age >= 18 ? 'Adult' : 'Minor';

// Вложенный тернарный оператор
const status = age >= 18
    ? 'Adult'
    : age < 18
        ? 'Child'
        : 'Nothing'
```

**Синтаксис:** `условие ? значение1 : значение2`
	по сути `?` = if,  `:` = else

### Операторы присваивания

`=`, `+=`, `-=`, `/=`, `*=`, `%=`, `**=`

```јс
d = d + 1; // == d += 1;
```

**Для строк `+=` это конкатенация:**

```js
let str = "Hello";
str += " world"; // "Hello world"
```

## 7. Классификация операторов

**Унарный оператор** - работает с одним операндом

```js
-x, +x, !x, typeof x, ++x, x--
```

**Бинарный оператор** - работает с двумя операндами

```js
a + b, a - b, a > b, a && b
```

**Тернарный оператор** - три операнда

```js
условие ? выражение1 : выражение2
```

## 8. Унарные плюс и минус

### Унарный минус (-)

Делает число отрицательным:

```js
const a = 10;
console.log(-a); // -10
```

### Унарный плюс (+)

Преобразует значение к числу:

```js
+'123'       // 123
+'3.14'      // 3.14
+true        // 1
+false       // 0
+'abc'       // NaN
```

То же самое, что `Number('123')`.

## 9. Логические операторы

### Оператор ИЛИ (`||`)

Возвращает **первое истинное значение / первое положительное thuthy значение  или последнее значение**

```js
// '' || null || false || 0 --> falsy values
const a = '' || null || false || 0 || 'Hi';
console.log(a); // 'Hi' (вернулся послдений операнд, потому что нет thuthy знаечний)
```

- много операторов лучше не писать, тк код будет не читаемым (несколькор операторов таких = окей)


### Оператор И (`&&`)

Возвращает **первое ложное значение** или последний операнд, если все истинные.

- при проверках возвращает true, если оба аргумента истины, а иначе -  false. или возвращает первое ложное знаечние как результат выражения
- вычисляет операнды слева направо. каждый операнд преобразует в логическое знаечние. если результат false, то останавливается и возвращает исхождное знаечние этого операнда. если все операнды были истинны, возвращается последний. // другими словами `&&` возвращет первое ложное знаечние, или послднее если нчиего не найдено.
	- разница в том что `&&` возвращает первое ложное значение а `ИЛИ` = первое истинное

```js
const a = 'Hi' && true && 5 && null;    // null
1 && 2 && 3                  // 3 (возвращает полсдений операнд тк все истинные)
10 && 0 && 20 && "Hello"; // 0 
```

- в react часто встречаетс ятакая запись с && (нпр `{isLoading && <div></div>}` - усли isLoading true то чтото отрисовываем)

**⚠️ Частая, но не рекомендуемая практика:**

```js
// ❌ Лучше использовать if
const age = 18;
const isYoung = age < 18;
isYoung && console.log('Close'); // если isYoung true, выполнится инструкция после &&
```

### Оператор НЕ (`!`)
- логическое отрицание, по стуи меняет значение на противоположное (сначала приводит аргумент к логическому типу парвда/ложь, затем возврщает противоположнео знаечние)
```js
let c  = !0; // 1) 0 - false 2) !fasle --> true
```


**Двойное отрицание (по сути это просто двойное применение оператора `!`):**

- приводит аргумент к логическому типу парвда/ложь (без реверса)

```js
!!0 === Boolean(0)   // true
// ложь = ложь
```

👉 **Но лучше использовать `Boolean()` - читабельнее**

### Приоритет операторов

`&&` имеет **приоритет выше**, чем `||`

```js
true || false && false
// то же самое, что true || (false && false)
```

## 10. Оператор нулевого слияния (`??`)

Возвращает **первое значение, которое НЕ null/undefined**.

- возвращает правый операнд если левый равен `null` или `undefined`, иначе возвращает левый операнд

```js
const res = null ?? 100; // 100
const res = false ?? 10; // false
const res = "hi" ?? 100; // "hi"
const res = undefined ?? 0; // 0
const res = null ?? 100; // 100
```

⚠️ `??` срабатывает **только** на `null` и `undefined`
