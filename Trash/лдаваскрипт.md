- `console.log(324) console.log("fdgfdg")` - выовдит чтото в консол разщработчика
- комменты: // или /\*\*/ - хороший код не нуждлается в комментариях! лрытный разрабы их редко юзают
- let, var, const
  `let userName = 'Mark'` - можем стаивть или не стаивть тчоку с запсятой (на выбор)
- типы перменных
  var - старый стандарт, не исопльузетися больше
  let - могут менять свое значение в ходе работы прогарммы
  const - перемнные под константы (не омгут менть значение во время работы програмым) // const p = 3.14;
  !изначально все перменные задаются через const а лишь потмо меняеются на let при неоьбходимости (если переменная все же будет изменяться) - более гибкая работа с перменными
- переменная константы должна быть сразщу инициализщирована в момент объясления перменной!
- лет и конст имеют область видимости внутри блока кода! 
  {} - объялвние блока кода . а вар имеют функциоанльную облатсь видимости, глобальную обьлатсь видимости
- типв данных: 
  1. строка let a = 'text'
  2. число let a = 30
  3. логичсекие значения let a = true
  4. null - ничего, пусто, неизвестно / используется чтобы сбросить какоето значение перменой
  5. undefined - обзанчеат что значение не было присвоено или оно не было определноо / возникает когда нпр объявили перемнную но не присвоили в нее какоето значение, те перменная обхъялвена но знаечние ще не уставнолена / +когда функция срабатывает но мы не рпописали что она явно чтото то она возвращет undefined // либо был совершен поиск элемента на страницу, нпр какотйо кнопки/тега и он не был найден то вернется undefined // возникает по ходу работы программы , но елси хотим обнулить то едлаем нулл
- в джс есть динамическая типизация (не нужно точно указывать тип перменной)
  let a = 'text' - строка
  a = 40- уде число
  a = труе - логичсекий тип
  не стоит этим зроупотреблять и использовать праивльный нейминг чтобы было понятно что это за перменная / не стоит менять значение рперемнной так резко, лучше даже создать новую перменную // имя перемнной должно быть в вижде существительного (нпр юзерНейм) // 
- let someName = 'James'; 
  someName = undefined; - так не делают
  someName = null; - так далают
- оператор typeof - похволяет определить тип перменной (мождет быть полезен с ложных преоктах)
  const someName = 'James'; 
  console.log(typeof someName);
  может бтыть полезен для отладки чужого кода
- BigInt - всроенный объект позволяющий создавать больше числа 
  Number.MAX_SAFE_INTEGER - свойство возвращающее максимальное значение Number типа, елси нужно число больше жэтого - используем BigInt // console.log(4385769843265874326587432687564328756874326528743265n) - используем BigInt вместо Number
  
  окгда работает с BigInt то все должно просиходитьт междлу BigInt офрматом нпр = console.log(4385769843265874326587432687564328756874326528743265n + 324324324) неправильно / console.log(4385769843265874326587432687564328756874326528743265n + 435435n) парвильно
