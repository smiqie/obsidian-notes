-  способы адаптации сайта - самые используемые:
	1. скрытие лишних блоков при уменшении экрана - которые не несут особой смысловой нагрзки (нпр декоративные изображения и разные украшательства), важные элеемнты тож можно скрывать но в таком случаех их важно помещать в какоето всплывающее меню открывающееся при клике на кнопку
	2. при уменьшении экрана зачастую перестраивают сетки (нпр за счет уменьшения колнок)
	3.  при уменьшении экрана уменьшают внешние и внутренние оступы 
	4. пропорционально ширине экрана уменьшают размер текста

- подходы адаптивной вертски:
	1. desktop-first: сначала верстается полноразмерная версия сайта а затем происзходит адаптация под мобилки
	2. mobile-first: снчала релаизуется версия сайта под экраны мобильных устрйств, затем делают адаптацию под гирокоформатные экраны
	- зачастую выбор подхода зависит от макета (если дизайнер подготовил исклчительно мобильную версию макетьа то логичнее юзать mobilefirst а потмо едлать дааптив для широких экранов --> пример). 
	- desktop first юзать проще!!!

- часто юзхаются для адаптива: `min/max-width/height`
- директива `@media` - позволяет указывтаь браузеру ксс правила которые будут применятся при опредленных параметрах экрана (размерах, плотности пикселей, типа устрйоства, пользоватлеьских настроек и многое другое).
```css
@media (параметры экрана --> max-width: 700px){
	правила для селекторов под более узкие экраны устройств
}
```

- тег `<meta name="viewport" content="width=device-width, user-scalable=no, initial-scale=1.0, maximum-scale=1.0, minimum-scale=1.0">`
	1. name="viewport" --> говорит о назначении мета тега
	2. content="" --> правила, о том, как следует обрабатывать старницу при разном мастшабировании при разных ширинах жэкранов
		- content="width=device-width -- самый главный параметр, ширина страницы = ширине экрана
		- user-scalable=no -- спорный но все же популряный параметр на отключении возможности  рользователя вручную мастшабирвоать страницу // лучше оставлять вклбченным!! иначе будет плозхая практика
		- initial-scale=1.0, maximum-scale=1.0, minimum-scale=1.0 -- задание коэфициентов масшабирвоания. начальный минимальный и максимальный везде по единицы
	3. если этого тега не будет, то медиа запросы не будут действовать на нашу страницу и фактичсеки интерейс станицы будет просто пропорционально мастшабироваться под экраны устрйоств

- пождкох mobile-first:
```css
@media (параметры экрана --> min-width: 401px){
	правила для селекторов под более узкие экраны устройств
}
```
- в медиа запросах также можем ориентирваотсья на выосту устрйоств (min/max-height)
```css
.banner-title{ color: black;}
@media (min-height: 900px){
	.banner-title{ color: green;}
}
@media (max-height: 400px){
	.banner-title{ color: red;}
}
```

- поплярный media запрос print - позволяет определить тип устрйоства - принтер на котрый выводится печать (нпр при нажатии соетании клавиш в хром ctrl+p то браузерп откроет превью страницы выводмиой на печать, к этой страницы будут применины все те стили которые прописаны в медиа запросы принт)
```css
@media print{
	body{
		font-size: 36px;
	}
	a{
		color: black;
		text-decoration: underline;
	}
}
```

```css
<button>Click</button>

/* на десктопной версии страницы все рабоатет правильно при наведении */
button{
	background-color: palegreen;
}

button:hover{
	background-color: crimson;
}

/* на тач устройствах код выше это плохй ux т.к. при задатие кнопке (не клике!) цвет покрасится в crimson и останется в таком состянии -- т.е. по сути залипнет, чтобы такого не просиходило модешь воспользоваться специальным медиа запрососм чтобы стили наведения применялись ток когда само наведении курсора впринципе возможно / другими словами чтобы ховер работал везде но не на тач устройвха */

button{
	background-color: palegreen;
}

/* конкретизировали css правило тко для нужных устройств у которых етсь курсор который возможно навести на элемент // т.е. эффект наведения на тач устровах уже не будет срабатывтаь */
@media (hover: hover) { 
	button:hover {
		background-color: crimson;
	}
}

/* !!! в DevTools по умолчанию в режиме эмулятора устройств все режимы по умолчанию являются тач устройствами (т.е. устройствами у которых нет мыши и взаимодействии с ними происходит за счет пальцев) !!! */
```


- помимо адаптивной верстки есть еще резиновая верстка - это подход рпи котором элементам задают размеры в относительных единицах измерения. это механика верстки сайтов с отзывчивым интерйефсом
- подходы резиной и адаптвиной верстки стоит применять в совокпуности



- готовые ширины экранов для медиа запросов:
  360-767 - мобилка
	768-1023 - планшет
	1024 и выше - десктоп
	
	 Диапазоны применимы к любому подходу. 
	 
	 Для desktop-first так: 
	 /* без медиазапроса - стили для десктопа */ 
	 @media (max-width: 1023px) { /* стили для планшета */ }
	 @media (max-width: 767px)  /* стили для мобилок */ } 
	 
	 Для mobile-first так: 
	 /* без медиазапроса - стили для мобилки */ 
	 @media (min-width: 768px { /* стили для планшета */ }
	  @media (min-width: 1024px) { /* стили для десктопа */ }
	или https://getbootstrap.com/docs/5.0/layout/breakpoints/
